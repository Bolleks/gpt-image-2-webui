# 🚀 Развертывание через Coolify

Эта инструкция поможет развернуть проект **GPT-Image-2-WebUI** на вашем сервере через **Coolify** с помощью Docker Compose. Следуйте шагам по порядку — всё сделано так, чтобы вам нужно было нажать минимум кнопок.

---

## 📋 Что вам понадобится

- **Сервер (VPS)** с Ubuntu 22.04+ (минимум 1 CPU, 2 GB RAM, 20 GB SSD)
- **Доменное имя**, направленное на IP вашего сервера (A-запись)
- **Git-репозиторий** с этим проектом (GitHub / GitLab / Gitea)
- **API-ключ KIE** — получите на сайте [KIE](https://kie.ai)

---

## Шаг 1: Установка Coolify на сервер

Подключитесь к серверу по SSH и выполните одну команду:

```bash
curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash
```

Установка займет 3–5 минут. В конце в терминале появятся:
- 🔗 **URL панели Coolify** (например: `http://123.456.789.0:8000`)
- 🔑 **Логин и пароль** для входа

Сохраните их! Зайдите по ссылке и авторизуйтесь.

> 💡 **Совет:** Сразу после первого входа Coolify предложит сменить пароль — сделайте это.

---

## Шаг 2: Добавление сервера в Coolify

1. В панели Coolify перейдите в раздел **Servers** (Серверы)
2. Нажмите **"+ Add Server"**
3. Введите:
   - **Name:** `My VPS` (или любое название)
   - **IP Address:** IP вашего сервера
   - **User:** `root`
   - **Port:** `22`
4. Coolify сам настроит Docker и все зависимости
5. Дождитесь статуса **"Online"** (зеленый индикатор)

---

## Шаг 3: Создание проекта

1. Перейдите в раздел **Projects** (Проекты)
2. Нажмите **"+ Add Project"**
3. Введите название: `gpt-image-webui`
4. Нажмите **Create**

---

## Шаг 4: Добавление ресурса (Docker Compose)

1. Внутри проекта нажмите **"+ Add Resource"**
2. Выберите тип: **Docker Compose** (обычно в категории "Application")
3. Выберите источник кода: **GitHub** (или ваш Git-провайдер)
4. Авторизуйте Coolify и выберите репозиторий с этим проектом
5. Ветка: `main` (или ваша основная ветка)
6. Нажмите **"Load Repository"**

Coolify автоматически найдет `docker-compose.yml` в корне проекта.

---

## Шаг 5: Настройка переменных окружения

Перейдите на вкладку **Environment Variables** (Переменные окружения) и добавьте:

| Переменная | Значение | Описание |
|------------|----------|----------|
| `APP_URL` | `https://your-domain.com` | Полный URL вашего сайта с `https://` |
| `AUTH_SECRET` | *(сгенерируйте команду ниже)* | Секретный ключ для сессий NextAuth |
| `AUTH_URL` | `https://your-domain.com` | URL для NextAuth (обычно = APP_URL) |

> ⚠️ **Важно:** `APP_URL` и `AUTH_URL` обязательно должны начинаться с `https://` и совпадать с вашим доменом.

Генерация AUTH_SECRET:
```bash
openssl rand -base64 32
```

---

## Шаг 6: Настройка домена

1. Перейдите на вкладку **Settings** (Настройки)
2. Найдите раздел **Domains**
3. Введите ваш домен: `your-domain.com`
4. Включите **HTTPS** — Coolify автоматически выпустит SSL-сертификат через Let's Encrypt
5. Сохраните настройки

> 🔒 Coolify сам настроит Traefik-прокси и SSL. Вам не нужно ничего делать вручную.

---

## Шаг 7: Деплой (запуск)

1. Нажмите большую зеленую кнопку **"Deploy"** (или **"Restart"**)
2. Coolify начнет:
   - Клонировать репозиторий
   - Собирать Docker-образ (это займет 3–5 минут в первый раз)
   - Запускать контейнер
   - Автоматически применять миграции базы данных
3. Перейдите на вкладку **Deployment Logs** и дождитесь сообщения:
   ```
   Ready on http://localhost:3000
   ```

🎉 **Готово!** Ваш сайт доступен по домену.

---

## Шаг 8: Первичная настройка приложения

После первого деплоя база данных пустая. При первом входе вам нужно будет создать аккаунт:

1. Откройте ваш домен в браузере: `https://your-domain.com`
2. Вас перенаправит на страницу **Регистрации**
3. Нажмите **"Создать аккаунт"** — вам будет показана seed-фраза из 12 слов
4. **Обязательно сохраните seed-фразу!** Она не будет показана снова
5. Скопируйте seed-фразу и нажмите **"Я сохранил — продолжить"**
6. После входа перейдите в раздел **Настройки** (иконка шестеренки)
7. Введите ваш **API-ключ KIE** и нажмите **Сохранить**

> 💡 **Seed-фраза** — это ваш единственный способ входа. Если вы её потеряете, восстановить доступ будет невозможно.

> 💡 **API-ключ KIE можно получить на сайте [KIE](https://kie.ai)**

> 💡 **Ваш ID** (для обращения в поддержку) отображается в боковой панели и на странице настроек

---

## Шаг 9: Проверка работы

1. Убедитесь, что страница загружается
2. Проверьте healthcheck: откройте `https://your-domain.com/api/health`
   - Должен вернуть: `{"status":"ok"}`
3. Попробуйте сгенерировать изображение — это проверит работу KIE API

---

## 🔄 Как обновить приложение

Когда вы внесете изменения в код и запушите их в репозиторий:

1. Зайдите в панель Coolify
2. Перейдите в ваш ресурс
3. Нажмите **"Pull Latest Changes"** (или включите **Auto Deploy** в настройках)
4. Coolify автоматически пересоберет и перезапустит приложение

> 💡 **Автодеплой:** В настройках ресурса включите **"Auto Deploy"** — тогда Coolify будет автоматически деплоить при каждом пуше в ветку.

## 📁 Что хранится в volume

Ваши данные (SQLite БД и изображения) сохраняются в Docker volume `app-data`:

- `/data/app.db` — база данных SQLite
- `/data/app.db-shm` и `/data/app.db-wal` — WAL-файлы SQLite
- `/data/images/` — сгенерированные изображения

При пересборке или обновлении контейнера эти данные **не теряются**.

## 🆘 Частые проблемы и решения

### Проблема: "Build failed" или ошибки сборки

**Решение:**
1. Откройте **Deployment Logs**
2. Найдите красную строку с ошибкой
3. Чаще всего это:
   - Отсутствует `package-lock.json` — убедитесь, что он запушен в репозиторий
   - Ошибка TypeScript — проверьте, что локально `npm run build` проходит без ошибок

### Проблема: "Application error" после деплоя

**Решение:**
1. Проверьте, что все обязательные Environment Variables заданы
2. Убедитесь, что `APP_URL` начинается с `https://`
3. Проверьте логи контейнера в разделе **Logs**

### Проблема: Изображения не генерируются

**Решение:**
1. Проверьте, что API-ключ KIE введен в **Настройках** приложения
2. Убедитесь, что ключ активен и на балансе есть кредиты
3. Проверьте логи приложения на наличие ошибок API

### Проблема: "Bad Gateway" или 502 ошибка

**Решение:**
1. Подождите 1–2 минуты — возможно, приложение еще запускается
2. Проверьте **Deployment Logs** — приложение может упасть с ошибкой
3. Убедитесь, что healthcheck проходит (см. логи)

## 🛠 Полезные команды (если нужно залезть внутрь)

Через SSH на сервере вы можете выполнять команды внутри контейнера:

```bash
# Посмотреть запущенные контейнеры
docker ps

# Зайти внутрь контейнера
docker exec -it gpt-image-2-webui sh

# Посмотреть логи приложения
docker logs -f gpt-image-2-webui

# Посмотреть размер volume с данными
docker volume ls
docker volume inspect gpt-image-2-webui_app-data
```
